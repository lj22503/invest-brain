# InvestBrain（brain.mangofolio.com）SEO 与 GEO 诊断与优化策略

> 诊断日期：2026-07-07 ｜ 站点：https://brain.mangofolio.com/
> 产品：InvestBrain — 投资第二大脑（mangoFolio 出品）
> 技术栈：Next.js 14 + Tailwind CSS ｜ MCP Server（FastMCP）+ SQLite 本地存储
> 定位：面向成熟投资者的「投资纪律 / 决策辅助」工具（镜子 + 纪律锚点），**不做投顾、不执行交易、数据本地存储**

---

## 一、诊断结论速览

InvestBrain 是三个 mangofolio 子站里**技术底子最好、也最有差异化护城河**的一个，但自然搜索流量同样接近于零。核心矛盾不是「做错了什么」，而是「该补的基础件还没补，且内容面太薄，没法承接搜索需求」。

### 技术健康评分

| 检查项 | 状态 | 说明 |
|--------|------|------|
| 渲染方式 | ✅ 良好 | Next.js 14 服务端渲染，正文内容在初始 HTML 中（搜索引擎可直接读取） |
| robots.txt | ✅ **优秀** | 已存在，且**明确放行 Googlebot / GPTBot / ChatGPT-User / Claude-bot / \***，并声明 Sitemap |
| sitemap.xml | ❌ **P0 缺失** | robots.txt 引用了 `https://brain.mangofolio.com/sitemap.xml`，但该 URL 返回 404 |
| meta description | ❌ 缺失（高置信） | 抓取结果未检测到 `<meta name="description">`，SERP 摘要将由 Google 随机截取，CTR 受损 |
| Open Graph 标签 | ❌ 缺失 | 社交分享 / 富媒体预览无图无摘要（og-image.png 尚未生成） |
| Twitter Card | ❌ 缺失 | 同上 |
| Canonical | ❌ 缺失（高置信） | 存在重复/参数页风险 |
| JSON-LD 结构化数据 | ❌ 缺失 | 丢失 FAQ / SoftwareApplication / Organization 富摘要机会 |
| llms.txt | ✅ 优秀 | 已存在，面向 AI Agent 的站点摘要完备 |
| /openapi/schema.json | ✅ 优秀 | 合法 JSON，MCP 工具清单完整——极强的 GEO / 开发者外链资产 |
| 索引状态 | ⚠️ 极低 | `site:`/品牌词搜索**未返回本站**，仅返回其它 mangofolio 子站（buddy / invest-kyc / mangoview），说明本站尚未被有效收录 |
| Google Search Console | ❌ 未注册 | 无法验证收录、提交 sitemap、排查问题 |

**一句话结论**：渲染与 robots 已就位，但 **sitemap 404 + 无 meta/OG/结构化数据 + 无内容矩阵 + 未注册 GSC** 四件事，让这个本可以「双向通吃传统搜索与 AI 搜索」的站点，目前两边都接不住。

## 二、相比 FinancePro / 雕龙，Brain 的独特优势（必须放大）

这是三站里**唯一一个同时为传统 SEO 和 GEO（生成式引擎优化）都做好了基建**的站点：

1. **AI 爬虫已放行 + llms.txt 就位** —— ChatGPT / Claude / Perplexity 已被允许抓取，且提供了面向 Agent 的标准化摘要。这是 2026 年最稀缺的「被 AI 引用」入场券，FinancePro、雕龙都不具备。
2. **内容叙事质量高** —— 「问题（市场每天喊你）→ 解决方案（投资锚三件事）→ 能力（6 大功能）→ 框架（10 步）→ FAQ」结构清晰，主题权威信号强。
3. **E-E-A-T 天然成立** —— 明确「不做投顾 / 不执行交易 / 数据本地」的产品原则，引用 16 位投资大师，GitHub 开源（lj22503/invest-brain），信任度高于大多数竞品。
4. **可链接资产丰富** —— `/openapi`（Skill 市场）、`/openapi/schema.json`（机器可读 MCP schema）、GitHub 仓库、飞书通知集成，都是开发者 / 极客圈天然会引用、会 Star 的资源。
5. **蓝海品类** —— 搜索「投资纪律工具 / 投资第二大脑 / 行为审计工具 / 投研教练」几乎没有直接产品竞品（竞品多为东方财富财富号、知乎、雪球的**文章**，而非产品）。Brain 可以「占品类词」。

## 三、关键词策略：概念领导力 + 品类占位 + GEO 引用

### 搜索意图分层

| 层级 | 代表关键词 | 月搜索量（估算） | 竞争度 | Brain 契合度 | 路线 |
|------|-----------|----------------|--------|-------------|------|
| 概念 / 教育（信息型） | 投资纪律、知行合一 投资、投资复盘、行为金融、追涨杀跌 行为偏差 | 中 | 低–中 | 极高（正中定位） | 博客 / 指南内容 |
| 独特资产（品牌/长尾） | 投资大师思想、巴菲特怎么看、芒格说、段永平思想 | 中 | 低 | 独特（大师思想库） | 资源页 / 引用 |
| 蓝海（几乎零竞争） | Socratic 投资教练、投研教练、投资第二大脑、投资纪律工具、行为审计 | 低–中 | ≈0 | 可占位 | 落地页 + 品类词 |
| 工具型（事务型） | 投资 AI 工具、投资思考工具、本地 投资 AI、MCP 投资 | 中–高 | 中–高 | 中 | 落地页 + 对比 |

### 高优先级目标词（建议首月主攻）

1. **投资纪律**（信息型，竞品全是文章）→ 写「投资纪律完全手册」「用规则对抗人性：投资纪律实操框架」
2. **知行合一 投资 / 知而不行**（信息型，2026 年活跃话题）→ 写「为什么道理都懂却做不到：投资中的知行合一」
3. **投资复盘**（事务/信息型，高意图）→ 写「投资复盘模板：如何记录决策逻辑而非盈亏数字」
4. **追涨杀跌 / 行为偏差**（信息型，Brain 行为模式报告直接对应）→ 写「追高、止损过早、割在最低点：用数据映照你的行为模式」
5. **投资大师思想 / 巴菲特怎么看XX**（独特资产）→ 大师思想库专题页 + 引用式内容
6. **投资纪律工具 / 投资第二大脑**（品类占位，≈0 竞争）→ 落地页直接吃下

### 内容缺口 vs 竞品

- 竞品（东方财富 / 雪球 / 知乎）只发**文章**，不发**工具**；Brain 是「文章级内容 + 可落地工具」的组合，应在每篇指南末尾引导使用对应 Skill（想法记录 / 行为模式报告 / 学习辅导）。
- Brain 独有「10 步框架」「Socratic 教练」「行为模式报告」——这些是**别人没有的方法论资产**，极适合做成「可收藏的终极指南」去抢 featured snippet。

## 四、GEO（生成式引擎优化）策略 —— Brain 的差异化主战场

因为 robots 已放行 AI 爬虫、llms.txt 已就位，Brain 应**把 GEO 当成头号战略**，而不是附属项：

1. **内容要「可被引用」**：每个核心概念页用清晰的「定义 + 要点列表 + 一句话结论」结构，方便 ChatGPT/Claude 直接摘录引用（这正是它们偏好的格式）。
2. **强化 llms.txt**：在 llms.txt 中补充各核心页面的「一句话价值主张 + 关键事实」，让 Agent 在回答「投资纪律工具有哪些」时优先带出 Brain。
3. **FAQ 结构化**：把现有 FAQ 扩成 8–10 条并加上 `FAQPage` JSON-LD，AI 概览常直接消费 FAQ 内容。
4. **被引用资产外推**：在 GitHub README、开发者社区（V2EX / 少数派 / 掘金）、AI 工具导航站发布「InvestBrain：让 AI 帮你守住投资纪律」类内容，争取被文章 / 列表引用（AI 训练语料多来自被引用文本）。
5. **schema.json 已是加分项**：保持 `/openapi/schema.json` 公开且可访问，开发者文章引用它时会把 Brain 带入技术语料。

## 五、技术修复清单（P0 → P1）

### P0（本周内，阻断收录）
1. **生成 sitemap.xml** —— Next.js 用 `app/sitemap.ts` 自动产出（落地页 + /openapi + 博客文章）；上线后 robots.txt 里的引用即生效。
2. **注册 Google Search Console + 百度站长平台**，提交 sitemap，申请收录。
3. **补 meta description**（≤160 字，含「投资纪律 / 投资第二大脑」核心词）。

### P1（2–4 周）
4. **补 Open Graph + Twitter Card**（需先有 og-image.png；可参考 FinancePro 同款 1200×630 图，建议为 Brain 单独出一张）。
5. **加 canonical**（避免参数/哈希页重复）。
6. **加 JSON-LD**：`Organization` + `WebSite` + `SoftwareApplication` + 首页 `FAQPage`。
7. **落地页增加结构化 FAQ**（承接「投资纪律 / 知行合一」类问题）。
8. **开通博客 / 指南中心**（`/blog` 或 `/learn`），承接信息型关键词（这是流量增长的主引擎）。

### P2（持续增长）
9. 内部链接：博客文章 ↔ 落地页 ↔ /openapi 互通，建「投资纪律」主题集群。
10. 跨站交叉链接：与 financepro / diaolong / buddy 互相引用（共享 mangofolio 域名权威）。
11. 开发者外链：GitHub README、MCP 目录（如 mcp.so、Smithery）、AI 工具导航站提交。

## 七、三阶段路线图

- **第 1–2 周（技术止血）**：上线 sitemap.ts、补 meta/OG/JSON-LD、注册 GSC/百度站长、提交收录。目标：首页 + /openapi 被收录。
- **第 1–3 月（内容 + GEO）**：开通博客，发布 15–20 篇指南（投资纪律 / 知行合一 / 复盘 / 行为偏差 / 大师思想），每篇带 FAQ + JSON-LD；扩 llms.txt；开发者社区投稿。目标：信息型词进入前 20，被 AI 概览引用。
- **第 3–6 月（权威 + 占位）**：进攻「投资纪律工具 / 投资第二大脑」品类词（≈0 竞争，直接冲前 3）；GitHub/导航站外链；跨站交叉链接。
