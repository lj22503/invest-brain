# 学习辅导框架融入 InvestBrain 设计文档

**日期**: 2026-06-26
**状态**: 已确认，待实施

## 一、目标

将「普通人如何建立自己的投资系统」框架内置进 InvestBrain，形成帮助普通投资者**自迭代认知**的完整闭环。

核心逻辑：强引导 + 零错误传播。用户获得的是结构化投研能力，不是随机答案。

## 三、新增：学习辅导模块

### 3.1 两档处理模式

**简单问题 → 直接10步输出**

触发条件（符合任一）：
- 单标的 + 简单方向（"茅台怎么看"、"该买黄金吗"）
- 单一变量（"美联储Q3降不降息"）
- 用户无困惑表达，只是询问

处理：一次性生成完整10步（见3.2），直接输出。

**复杂主题 → Socratic多轮引导**

触发条件（符合任一）：
- 问"为什么"或"为什么不一样"（含比较）
- 多资产联动（"黄金和石油为什么走势不同"）
- 用户表达困惑或错误认知（"我觉得美元要见顶了"）
- 政策+市场交叉（"鲍威尔转鸽对A股科技什么影响"）

处理：分轮对话，逐步引导，暴露用户认知gap，每轮给出选项。

### 3.2 第一组：基础投研核心Prompt（10步标准模板）

用户提问后，直接一次性生成完整10步输出（简单模式）或作为多轮对话的每轮内容基础（复杂模式）。

```
1. 请拆解【输入具体宏观事件】影响的关键宏观/市场变量，并按照对资产价格的影响重要性排序

2. 请把上述变量的相互作用和传导逻辑，用带箭头的因果链完整表达出来

3. 上述变量的变化，会分别如何影响利率、汇率、A股/美股、大宗商品四类资产的价格趋势？请分品类说明

4. 历史上是否出现过和当前变量结构一致的情景？请列出对应时间和最终各类资产的表现

5. 情景推演：如果【变量A发生XX变化】但【变量B没有出现预期的跟随变化】，这意味着什么？会对原有传导逻辑产生什么改变？

6. 当前市场（最近1-2周）价格变动，最隐含的核心交易逻辑是什么？这个逻辑和公开新闻、主流观点一致吗？如果不一致，差异在哪里？

7. 如果上述推演逻辑成立，请给出具体可落地的交易方案：明确交易标的、方向、核心支撑逻辑，以及大致的持仓时间窗口

8. 反向校验：如果当前这个交易逻辑是错的，最可能的漏洞在哪里？哪一个变量的变化会证伪这个逻辑？

9. 请明确写出这个交易逻辑的失效条件：当出现哪些信号/变量变化时，必须离场，这个逻辑已经不成立了？

10. 请用3句话总结本次分析：第一句讲当前市场核心结构，第二句讲核心驱动变量，第三句讲交易方向，用于归档笔记
```

### 3.3 第二组：系统搭建Prompt（对应记忆/触发/反馈三层）

这组 Prompt 用于搭建和维护用户专属的投研系统，**按需触发，不是每次对话都调**。

| Prompt | 触发时机 | 调用方式 |
|--------|---------|---------|
| **2.1 新事件复盘归档** | 每次辅导/分析结束后 | **自动调用**，结果写入情景库 |
| **2.2 新事件对比历史** | 用户问"和XX事件比有什么不同"时 | 用户触发，从情景库检索相关事件后调用 |
| **2.3 触发阈值建议** | 用户问"XX多少算触发"时；或每月定时 | 用户触发 + 定时触发（提醒） |
| **2.4 判断函数迭代** | 用户说"我上次判断错了"时 | 用户触发，从情景库取错误记录后调用 |

**2.1 新事件复盘归档（每次辅导结束自动调用）**
```
请把这次分析的以下内容整理成结构化条目，存入我的情景案例库：
- 触发事件：【具体描述】
- 核心变量结构：【变量列表】
- 推演情景：【预判的因果链】
- 实际市场结果：【最终走势】
- 失效点/正确点：【哪里对哪里错】
```

**2.2 新事件对比历史（用户问及时触发）**
```
请帮我对比当前这个新事件的变量结构，和我情景库中【XX年XX事件】的结构有什么异同？
- 相同点是什么？
- 核心差异是什么？
- 上次事件的教训是否适用于本次？
```

**2.3 触发阈值建议（用户问或每月定时触发）**
```
根据我过去1年的交易记录，帮我总结：
对于【利率/汇率/商品】，多大的变动幅度应该触发深度分析？
请给我具体的数值阈值建议，并说明理由。
```

**2.4 判断函数迭代（用户说上次判断错了时触发）**
```
这次交易判断错误：
- 错误点是：【XX变量】没发生预期变化
- 正确结果是：【XX】

请帮我更新我的投研规则：下次遇到类似结构，应该把哪个变量优先级调高，哪个调低？
```

### 3.4 第三组：每日最小执行Prompt

用户每日固定使用，快速完成日常投研流程。

**触发方式：**
- **提醒触发**：每日提醒 → 用户点开 → 执行
- **用户主动**：用户来问"今天市场有什么要分析的吗"

```
今天触发我分析的事件是【XX】，请帮我按照以下步骤完成今日投研：

1. 拆解核心变量并排序
2. 画出传导因果链
3. 对比历史情景
4. 推演两种核心情景，写出对应交易方案
5. 明确写出失效条件和止损规则

最后用3句话总结归档。
```

### 3.3 信息正确性保障（A+B+D原则）

**A. 自动校验**
- 大师思想/宏观数据输出必须带来源引用
- RAG结果与知识图谱交叉验证，不一致时提示"有争议"
- 情景库写入前，核心因果链需用户确认

**B. 用户最终把关**
- 所有结论明确标注"仅供参考，不构成投资建议"
- 每步结论由用户自己判断，不替用户做决策
- 用户可修改/否定系统给出的因果链

**D. 限制引导范围**
- 确定性高的场景（宏观事件、大类资产）触发强引导
- 不确定或争议大的问题，降级为普通RAG，不强引导
- 涉及具体荐股/择时，不触发辅导模式

## 四、新增数据表：情景库

在现有SQLite memory库中新增表：

```sql
CREATE TABLE scenarios (
    id TEXT PRIMARY KEY,
    trigger_event TEXT NOT NULL,        -- 触发事件描述
    variable_structure TEXT,             -- 核心变量结构（JSON）
    causal_chain TEXT,                   -- 因果链描述
    predicted_outcome TEXT,              -- 预判结果
    actual_outcome TEXT,                 -- 实际结果
    lesson TEXT,                        -- 一句话教训
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

ALTER TABLE thoughts ADD COLUMN scenario_id TEXT REFERENCES scenarios(id);
```

情景库与想法体系的关系：
- 想法可独立存在（scenario_id = null）
- 想法可归属某情景（scenario_id 关联）
- 查情景库时可拉出相关历史想法
- 同一事件的辅导结果和想法记录可交叉引用

## 五、认知迭代闭环

```
用户提问
  ↓
系统判断：简单 or 复杂
  ↓
【简单】→ 直接10步输出
【复杂】→ Socratic多轮引导，暴露认知gap
  ↓
两者结果写入情景库
  ↓
下次同类问题 → 调出情景库对比历史
  ↓
用户经历"预判 vs 实际" → 更新判断函数
  ↓
时间积累 → 认知越来越接近实际
```

认知提升的两个主要机制：
1. **情景库积累**：历史情景多了，用户能借鉴，认知变深
2. **Socratic追问**：每次辅导戳认知gap，用户被迫想清楚
