# 投资助手 — SPEC.md

## 1. 概述

### 1.1 项目背景

**核心问题**：普通投资者"知行合一"问题 — 被叙事带着走，操作没纪律，事后后悔

**定位**：投资界的"第二大脑"，被其他AI Agent调用的Skill/MCP服务

**核心价值**：让投资者"越用越聪明" — 不是投顾，不给建议，而是"镜子"（让你看清自己）+ "纪律锚点"（对照大师思想检查自己的行为）

### 1.2差异化

| 竞品 | 核心功能 | 我们 |
|------|----------|------|
| WindClaw/同花顺 | 数据+交易执行 | 认知迭代+决策复盘+思维模型 |
| 投顾机器人 | 给建议 | 不给建议，给"拷问" |
| Get笔记/得到大脑 | 知识管理 | 专注投资纪律+行为审计 |

### 1.3 目标用户

- 个人投资者（第一个用户：我自己）
- 有一定投资经验但控制不住情绪、缺乏纪律
- 熟悉AI Agent交互方式

## 2. 功能范围

### 2.1 MVP四大功能

#### 功能1：想法记录

**输入**：用户说一句话（文字/语音转文字）

**处理**：
- AI解析：识别标的（茅台）、价格（1400）、指标（PE）、时间对比
- 关联历史：搜索用户历史中关于同一标的的记录
- 关联大师：检查用户想法是否与某位大师原则矛盾
- 生成卡片：结构化输出

**输出**：想法卡片，包含：
- 原始输入
- AI解析的实体
- 历史关联（相关决策、价格、估值）
- 大师对照（如有）
- 可选：自动创建提醒

**触发**：用户主动说/输入

---

#### 功能2：投资知识RAG

**输入**：用户问投资问题

**处理**：
- 检索大师思想库（巴菲特、芒格、段永平等）
- 检索投资理论（价值投资、周期理论等）
- 结合用户上下文（记忆系统）生成个性化回答

**输出**：带来源引用的回答，可附加"拷问问题"

**触发**：用户主动问

---

#### 功能3：记忆系统

**数据存储**：
- 用户画像（风险偏好、能力圈、投资原则）
- 历史决策（买了什么、为什么买、什么时候卖）
- 行为模式（追高、止损过早等，自动识别）

**能力**：
- 语义检索（"我上次关于茅台的想法是什么？"）
- 模式发现（"你最近3次买入都是在PE分位<15%时"）
- 矛盾提醒（"你去年说不再追高，但这次..."）

**数据位置**：本地存储

---

#### 功能4：提醒触发

**触发条件**：
- 价格触发（"茅台跌破1400提醒我"）
- 时间触发（"每周五检查持仓"）
- 条件触发（"当PE分位低于20%时提醒我"）

**执行**：定时检查条件，满足时通知用户

**通知方式**：可通过用户已有的Agent推送

---

### 2.2 大师思想库（预置）

覆盖大师：
- 巴菲特（护城河、安全边际、能力圈）
- 芒格（多元思维模型、好生意）
- 段永平（本分、不懂不投）
- 霍华德·马克斯（周期、风险）
- 李录（文明现代化价值投资）

知识组织：知识图谱Schema
- 实体类型：大师、理论、策略、市场条件
- 关系类型：继承自、矛盾于、适用于

---

### 2.3 圆桌讨论模式（扩展功能，V2）

**输入**：用户的投资困惑

**处理**：
- 邀请3-5位大师角色
- 多轮辩论，可互相反驳
- 用户可中途追问

**输出**：对话流文本 + 每位大师核心结论

## 3. 技术方案

### 3.1 架构模式

**Expert Pool（专家池）模式**

```
主Agent（调度）
├── 想法记录Skill
├── 投资知识RAG Skill
├── 记忆Skill
├── 提醒Skill
└── 大师Skill（多个）
```

每个Skill是独立可调用的，主Agent根据用户意图调用相应Skill。

### 3.2 技术选型

| 组件 | 选型 | 理由 |
|------|------|------|
| Agent框架 | Harness | 你已有的框架，多Agent协作成熟 |
| 知识存储 | 混合（Graph + Vector） | 投资知识有因果关系，GraphRAG更合适 |
| 记忆存储 | 本地SQLite + Chroma | 本地优先，隐私好 |
| 数据源 | AKShare（免费）+ MCP扩展 | 优先免费，允许用户补充收费API |
| LLM | DeepSeek V4 系列 | Pro（复杂推理）+ Flash（快速调用）|
| 交付形式 | MCP Server + Skill包 | 同时支持两种调用方式 |

### 3.3 知识图谱Schema

**实体类型**：
- `投资大师`：name, 背景, 核心理念, 方法论, 适用条件, 局限
- `投资理论`：所属门派, 核心逻辑, 与其他理论的关系
- `投资策略`：触发条件, 操作方式, 风险边界
- `市场条件`：牛市/熊市/衰退, 估值水平, 情绪指标

**关系类型**：
- `继承自`（巴菲特→段永平）
- `发展自`（护城河→竞争壁垒）
- `矛盾于`（价值投资 vs 趋势投资）
- `适用于`（价值投资→优质蓝筹）
- `警示`（不要加杠杆）

### 3.5 LLM 集成配置

#### 3.5.1 模型规格

| 模型 | 模型ID | 输入价格 | 输出价格 |
|------|--------|---------|---------|
| DeepSeek-V4-Pro | `deepseek-ai/deepseek-v4-pro` | $1.74/M | $3.48/M |
| DeepSeek-V4-Flash | `deepseek-ai/deepseek-v4-flash` | $0.14/M | $0.28/M |

#### 3.5.2 场景 × 模型 × 温度

| 场景 | 模型 | 思考模式 | 温度 | 理由 |
|------|------|---------|------|------|
| 想法解析 | V4-Flash | 关闭 | 0.3 | 实体提取要准确，不随机发挥 |
| RAG合成 | V4-Flash | 关闭 | 0.5 | 连贯 + 适度创造性 |
| 行为模式分析 | V4-Pro | 开启（reasoning_effort=高） | 0.4 | 深度推理，但不天马行空 |
| 圆桌讨论-角色话术 | V4-Flash | 关闭 | 0.7 | 角色风格要鲜明，适度随机 |
| 圆桌讨论-综合裁判 | V4-Pro | 开启（reasoning_effort=高） | 0.5 | 裁判需要推理但不极端 |

#### 3.5.3 API Key 配置

用户只需提供 `DEEPSEEK_API_KEY`，模型路由和温度由代码内部处理。

### 3.4 数据存储结构

```
data/
├── knowledge/ # 大师思想知识库
│   ├── graph/          # 知识图谱（neo4j或sqlite）
│   └── vectors/        # 向量索引（chroma）
├── memory/             # 用户记忆
│   ├── user_profile/   # 用户画像
│   ├── decisions/      # 历史决策
│   └── patterns/       # 行为模式
├── cards/              # 想法卡片
└── reminders/          # 提醒条件
```

## 4. 接口设计

### 4.1 MCP Server接口

```json
// 想法记录
tools: [
  "record_thought(text)", // 记录想法
  "search_memories(query)",    // 检索记忆
  "get_thought_cards(ticker)", // 获取某标的想法卡片
]

// 投资知识RAG
tools: [
  "ask_investment(question)", // 投资问答
  "get_master_view(master, topic)", // 获取某大师对某话题的看法
  "search_knowledge(query)",         // 知识检索
]

// 记忆系统
tools: [
  "get_user_profile()",        // 获取用户画像
  "record_decision(data)",     // 记录决策
  "get_behavior_patterns()",   // 获取行为模式
]

// 提醒
tools: [
  "set_reminder(condition)",   // 设置提醒
  "get_reminders()",           // 获取所有提醒
  "delete_reminder(id)",       // 删除提醒
]
```

### 4.2 Skill包结构

```
investment-skill/
├── SKILL.md              # Skill定义
├── src/
│   ├── record_thought/
│   ├── investment_rag/
│   ├── memory/
│   ├── reminder/
│   └── masters/ # 大师Skill
└── data/                 # 内置知识库
```

## 7. 不做什么

- 不做交易执行
- 不做实时行情数据库（依赖第三方）
- 不做移动端App（纯服务，被Agent调用）
- 不限制用户使用方式（不锚定五步法）
