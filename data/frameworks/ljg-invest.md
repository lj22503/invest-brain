# 秩序机 — 投资分析框架

生成投资分析报告。核心只问一个问题：这个东西在创造新秩序，还是在搬运旧秩序？

## 认知起点

财富不是钱，是被欲望照亮的秩序。投资就是拿手里的秩序去换一台更好的秩序生成器。

所以不称重，看相：
- 不问"这个公司值多少钱"，问"这台机器转不转得起来"
- 不问"市场多大"，问"市场在用什么过时的眼睛看它"
- 不问"能涨多少"，问"我拿什么换什么，换完之后谁更聪明"

## 核心判定：秩序创造机器

从三个角度看：

**飞轮在不在转？**
系统有没有越用越好的结构——用户越多数据越多，数据越多产品越好，产品越好用户越多？这个循环是已经在转、刚开始、还是停着？

**冲击后变强还是变弱？**
竞争来了、技术变了、市场塌了——这台机器是碎掉、扛住、还是吃掉冲击变成自己的燃料？

**资源是被推过来的，还是自己来的？**
扩张靠的是一个一个谈、一块一块买（推），还是别人主动涌过来、因为不来就亏（引力）？

**综合判定**：
- **秩序创造机器** — 飞轮在转，冲击后变强，资源自己来
- **有潜力** — 飞轮有结构但还没验证转起来
- **秩序搬运** — 把已有的东西重新排列，没有生成新秩序

## 报告结构

### 一、这是什么

一张表 + 一句自定义赛道定义。

| 维度     | 内容                                         |
|----------|----------------------------------------------|
| 项目名称 |                                              |
| 赛道定义 | 用我们自己的语言，不用市场标签               |
| 阶段     |                                              |
| 融资情况 | 金额 / 估值 / 条款                           |
| 数据快照 | 关键运营数据                                 |

赛道定义要穿透表面标签。"搜索引擎公司"是隔的，"人类认知基础设施的垄断运营商"是不隔的。

### 二、秩序创造机器判定

**飞轮在不在转？**
- 用户越多 -> 数据越多 -> 产品越好 -> 用户越多
- 循环是否已经验证？加速还是匀速？

**冲击后变强还是变弱？**
- 历史上有无被冲击案例？结果如何？

**资源是自己来的还是推来的？**
- 有没有"不推而聚"的迹象？

### 三、创生公式

每台秩序创造机器都有一个核心算法。

参考：
- 亚马逊 = 利润->再投资->降成本->降价->更多用户->更多利润
- 特斯拉 = 硬件采数据->数据训练算法->算法重新定义硬件
- Google = 每次人类找答案的方式迁移时，成为新方式的默认基础设施

### 四、市场看见的 vs 我们看见的

**它在 S 曲线的哪里？**
积累期、拐点、加速期、平台期。

**市场在用什么旧眼睛看它？**
市场给它贴的标签是什么？这个标签遮住了什么？

三个信号检测认知折价：
- 需要很复杂的解释别人才能听懂？
- 定价持续异常（部分之和 != 整体）？
- 已有的类比全部失效？

**它控制了什么别人拿不走的东西？**
数据、分发、标准、网络效应？静态（品牌、专利）还是动态（越变越强）？

### 五、换不换

**交换建议**：建议投资 / 建议观察 / 建议放弃

**核心假设**：决策依赖哪几个假设？每个假设附退出信号。

**未解问题**：3-5个对决策至关重要但还没答案的问题。
