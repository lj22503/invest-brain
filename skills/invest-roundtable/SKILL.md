## Usage

```
/invest-roundtable 贵州茅台现在估值合理吗？
/invest-roundtable 美联储降息背景下美股科技股还能追吗？
/圆桌 宁德时代 vs 比亚迪，谁更值得长期持有？
```

## Instructions

### 一、定议题

用户给了议题直接用。只说「圆桌」或「大师圆桌」没带议题，先问一句。

### 二、匹配大师

调用 MCP 工具 `invest_roundtable`，传入用户议题。工具内部会：

1. 将议题向量化，在 16 位大师池中做语义匹配
2. 返回 3-5 位最相关的大师（含 MBTI、立场、入选理由）
3. 以主持人口吻开场，介绍议题和与会大师

### 三、发言格式

全场统一格式：

```
【大师名】【行动标签】：发言内容

**简言之**：一句话总结
```

六种行动标签：`陈述` `质疑` `补充` `反驳` `修正` `综合`

### 四、讨论规则

**主持人（系统）职责**：
- 开场抛出定义问题，让大师们先对齐核心概念
- 每轮追一个争议点，不摊大饼
- 大师们说得客气时，点破分歧摆回桌面
- 每轮后做综述：核心争议 + 立场分布 + 下一层问题

**大师发言要求**：
- 忠于本人真实思想体系，引用其经典言论和方法论
- 必须接着前面的话说（质疑/补充/反驳），不许自说自话
- 泛泛而谈的正确废话不要

### 五、输出结构

讨论结束后，工具返回结构化 JSON，AI 据此生成 Markdown 纪要：

```markdown
## 圆桌：{议题}

### 与会大师
| 大师 | MBTI | 立场 | 入选理由 |
|------|------|------|----------|
| ... | ... | ... | ... |

### 开场：定义对齐
[主持人开场词 + 各大师对核心概念的定义发言]

### 讨论交锋（逐轮）
#### 第 N 轮：{引导问题}
[各大师发言，含行动标签和简言之]
[主持人综述：争议点 + 下一层问题]

### 终局判断
- **共识**：大师们一致认同的结论
- **分歧**：核心分歧及各自前提假设
- **拷问清单**：讨论中浮现但未深挖的关键问题
```

### 六、存档

讨论全文写入项目 `data/cards/` 目录下的 Markdown 文件：

```
data/cards/圆桌-{议题关键词}-{YYYYMMDD}.md
```

写完把文件路径报给用户。
